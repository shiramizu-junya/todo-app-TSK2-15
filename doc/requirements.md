# TODOアプリケーション MVP要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
TODOアプリケーション

### 1.2 目的
個人のタスク管理を効率化するためのWebアプリケーションを開発する。

### 1.3 対象ユーザー
個人利用（1ユーザー想定）

### 1.4 MVPのゴール
基本的なタスク管理機能を備えた、シンプルで使いやすいTODOアプリケーションをリリースする。

## 3. 機能要件

### 3.1 認証機能

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| AUTH-001 | ユーザー登録 | メールアドレスとパスワードで新規登録 | 必須 |
| AUTH-002 | ログイン | メールアドレスとパスワードでログイン | 必須 |
| AUTH-003 | ログアウト | セッションを終了してログアウト | 必須 |
| AUTH-004 | パスワードリセット | メールでパスワードリセットリンクを送信 | 必須 |

### 3.2 TODO管理機能

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| TODO-001 | TODO作成 | 新しいTODOを作成する | 必須 |
| TODO-002 | TODO一覧表示 | TODOをリスト形式で表示する | 必須 |
| TODO-003 | TODO詳細表示 | TODOの詳細情報を表示する | 必須 |
| TODO-004 | TODO編集 | 既存のTODOを編集する | 必須 |
| TODO-005 | TODO削除 | TODOをゴミ箱に移動する | 必須 |
| TODO-006 | ステータス変更 | TODOのステータスを変更する | 必須 |

### 3.3 フィルター・検索機能

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| FILTER-001 | ステータスフィルター | ステータスでTODOを絞り込む | 必須 |
| FILTER-002 | キーワード検索 | タイトル・詳細でTODOを検索する | 必須 |

### 3.4 ゴミ箱機能

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| TRASH-001 | ゴミ箱一覧表示 | 削除したTODOを一覧表示する | 必須 |
| TRASH-002 | TODO復元 | ゴミ箱からTODOを復元する | 必須 |
| TRASH-003 | 完全削除 | ゴミ箱からTODOを完全に削除する | 必須 |

## 4. データ定義

### 4.1 TODOデータ項目

| 項目名 | データ型 | 必須 | 説明 |
|--------|----------|------|------|
| id | UUID | ○ | 一意識別子 |
| user_id | UUID | ○ | 所有ユーザーID |
| title | VARCHAR(100) | ○ | タイトル（最大100文字） |
| description | TEXT | - | 詳細・メモ |

| 値 | 表示名 |
|----|--------|

## 6. 画面詳細

### 6.1 ログイン画面（SCR-001）

**入力項目**
- メールアドレス（必須）
- パスワード（必須）

**アクション**
- ログインボタン → 認証成功でTODO一覧画面へ遷移
- 新規登録リンク → 新規登録画面へ遷移
- パスワードを忘れた方リンク → パスワードリセット画面へ遷移

### 6.2 新規登録画面（SCR-002）

**入力項目**
- メールアドレス（必須）
- パスワード（必須、8文字以上）
- パスワード確認（必須）

**アクション**
- 登録ボタン → 登録成功でTODO一覧画面へ遷移
- ログインリンク → ログイン画面へ遷移

### 6.3 TODO一覧画面（SCR-004）

**表示内容**
- TODOリスト（タイトル、期限、優先度、ステータス、カテゴリ）
- フィルターエリア（ステータス選択）
- 検索バー

**デフォルト表示**
- 完了ステータスのTODOは非表示
- 削除済み（ゴミ箱）のTODOは非表示

**アクション**
- TODO新規作成ボタン → TODO作成画面へ遷移
- TODOクリック → TODO詳細画面へ遷移
- ステータス変更 → インラインでステータス変更
- ゴミ箱アイコン → ゴミ箱画面へ遷移
- ログアウトボタン → ログアウト処理

### 6.4 TODO作成画面（SCR-005）

**入力項目**
- タイトル（必須、最大100文字）
- 詳細・メモ（任意）
- 期限（任意、日付選択）
- 優先度（必須、デフォルト：中）
- カテゴリ（必須、デフォルト：その他）
- 画像（任意、1枚まで）

**アクション**
- 保存ボタン → TODO作成してTODO一覧画面へ遷移
- キャンセルボタン → TODO一覧画面へ遷移

### 6.5 TODO詳細画面（SCR-006）

**表示内容**
- 全てのTODO項目を表示
- 添付画像がある場合は画像表示

**アクション**
- 編集ボタン → TODO編集画面へ遷移
- 削除ボタン → 確認後、ゴミ箱へ移動してTODO一覧画面へ遷移
- 戻るボタン → TODO一覧画面へ遷移

### 6.6 TODO編集画面（SCR-007）

**入力項目**
- TODO作成画面と同じ項目（既存データが初期値）

**アクション**
- 保存ボタン → TODO更新してTODO詳細画面へ遷移
- キャンセルボタン → TODO詳細画面へ遷移
- 画像削除 → 添付画像を削除

### 6.7 ゴミ箱画面（SCR-008）

**表示内容**
- 削除済みTODOリスト（タイトル、削除日時）

**アクション**
- 復元ボタン → TODOを復元してTODO一覧画面へ遷移
- 完全削除ボタン → 確認後、TODOを完全削除
- 戻るボタン → TODO一覧画面へ遷移

## 7. 非機能要件

### 7.1 セキュリティ
- Supabase Row Level Security（RLS）によるデータアクセス制御
- 認証ユーザーのみが自身のTODOにアクセス可能

### 7.2 レスポンシブ対応
- モバイル、タブレット、デスクトップに対応

### 7.3 画像アップロード
- 対応形式：JPEG, PNG, GIF, WebP
- 最大ファイルサイズ：5MB
- Supabase Storageに保存

## 8. MVPスコープ外（将来の拡張候補）

以下の機能はMVPには含めず、将来のバージョンで検討する。

- 通知・リマインダー機能
- カンバン表示（Trello風）
- ドラッグ＆ドロップによる並び替え
- TODOの複製機能
- タグ機能（複数カテゴリ）
- 繰り返しタスク
- サブタスク
- ダークモード
- データエクスポート
